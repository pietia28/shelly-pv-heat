2+2 sensor split: max 3 DS18B20 per Add-on is reliable (with 4, the 85 °C error appeared). Four sensors split two-per-device stay well within the limit.

How it works

The master (1PM Gen4) reads temperatures and grid power every 30 s and makes decisions. It reads the buffer and DHW 1 locally, pulls DHW 2 and DHW 3 from the module over RPC, and the surplus from the Pro 3EM.

Flexible sensor configuration

Sensor placement is described by a single block in the script. Each sensor has a local flag (master vs module) and an id. Changing the split (e.g. 1+3 or 3+1) is just flipping flags — the control logic stays the same.

// local:true = on MASTER, local:false = on MODULE (RPC)
let SENSORS = [
  { key: "buffer", local: true,  id: 100 },  // buffer  (MASTER)
  { key: "cwu1",   local: true,  id: 101 },  // DHW 1   (MASTER)
  { key: "cwu2",   local: false, id: 100 },  // DHW 2   (MODULE)
  { key: "cwu3",   local: false, id: 101 }   // DHW 3   (MODULE)
];

Safety layers

Buffer over-temperature cutoff

All heaters switch off at the buffer maximum, regardless of surplus; they resume after a hysteresis drop. The buffer is local = always available.

Heater watchdog

Each stage has auto-off (~90 s), refreshed every cycle. If the master goes silent, the heaters switch off by themselves.

Measurement fail-safe

No reading from the 3EM = heaters OFF.

85 °C filter + last-good

A DHW reading of exactly 85 °C (DS18B20 error) is rejected; the script keeps the last good value.

Prerequisites

  • All devices on the same LAN, up-to-date firmware.
  • Static IPs for the 5 remote devices (module, 3EM, 3× Pro 3).
  • RPC authentication disabled on the LAN (the script calls /rpc without login).
  • Sensors: 2 DS18B20 per Add-on (max 3), daisy-chained, with one 4.7 kΩ pull-up (3.3 kΩ if problems) per bus.

Commissioning step by step

1

IP addressing

  1. Assign static IPs: the module, Pro 3EM, and the three Pro 3.
  2. Enter them into CFG: TEMPDEV_IP, EM_IP, HEATER1_IP, HEATER2_IP, HEATER3_IP.
2

DS18B20 sensors — two devices, 2 each

  1. Master (1PM Gen4): Add-on + 2 sensors — buffer + DHW 1.
  2. Module (1PM Gen4): Add-on + 2 sensors — DHW 2 + DHW 3.
  3. On both: Peripherals → Temperature (DS18B20) → Rescan. Chain + one 4.7 kΩ pull-up per bus (see the separate wiring diagram).
Note: do not exceed 3 sensors per Add-on — this caused the 85 °C errors.
3

Identify sensors (by id)

Newer firmware does not expose the 1-Wire address — sensors are referenced by component id. Note: ids start at 100 separately on each device (master 100/101, module 100/101).

  1. Check a sensor value: curl "http://<IP>/rpc/Temperature.GetStatus?id=100" → returns tC.
  2. Physical identification: warm one sensor by hand, query again, and see which id's tC jumped.
  3. Enter your findings into the SENSORS block: the local flag (master/module) + id for each sensor.
ids may be renumbered after a full Rescan — re-check the mapping after each rescan.
4

Pro 3EM — surplus metering

  1. 3-phase profile, CTs on the main circuit past the grid connection.
  2. Verify after start: the console line Nadwyzka: … W (surplus) must be positive when exporting. If negative — change EXPORT_SIGN from -1 to +1.
5

Pro 3 — heaters

Confirm that channels id 0/1/2 are stages 1/2/3. The script sets Auto-OFF itself at start.

6

Upload the script (on the MASTER)

  1. On the 1PM Gen4: Scripts → Add script, paste sterowanie_grzalki_pompa_v5.js.
  2. Fill in CFG (IPs, EXPORT_SIGN, BUFFER_MAX, CWU_MAX) and the SENSORS block (local + id).
  3. Save → Start, enable Run on startup.
  4. Power-up order: remote devices first (module, 3EM, Pro 3), then the master.
7

Commissioning tests

  • Pump ON/OFF — buffer − DHW ≥ 10 °C → ON; below 6 °C → OFF.
  • Anti-scald — DHW to CWU_MAX → pump OFF.
  • Heaters — surplus rises → stages one by one; force a load → they shed, no grid import.
  • Buffer limit — buffer to BUFFER_MAX → heaters OFF.
  • Watchdog — Stop the script → heaters switch off in ~90 s.
  • 3EM fail-safe — wrong EM_IP → heaters OFF.
  • Module offline — disconnect it: DHW 2/3 hold their last value, heaters keep running (buffer is local).

Sensor diagnostics 85 °C error

85.0 °C is not a temperature — it is the DS18B20 power-up value on failed communication. If a sensor shows exactly 85 °C, check in order:

  • Sensor count — max 3 per Add-on (VCC output limited to 10 mA).
  • Pull-up — one 4.7 kΩ resistor (3.3 kΩ if problems) DATA↔VCC per bus, close to the Add-on.
  • Topology and cable — chain, not star; adequate cable cross-section, away from power wiring.
  • Hardware — if all still show 85 despite the above, suspect a faulty Add-on or poor-batch sensors.

Parameters to agree

  • BUFFER_MAX — buffer maximum temperature (set below 85 °C).
  • CWU_MAX — anti-scald DHW limit (0 = off).
  • Pump thresholds PUMP_ON_DIFF / PUMP_OFF_DIFF — default 10 / 6 °C.
